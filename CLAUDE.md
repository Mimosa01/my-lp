# Контекст проекта

Ты помогаешь разрабатывать быстрые лендинги для самозанятых (мастера, репетиторы, услуги).

Цель сайтов:
- показывать услуги и портфолио
- отвечать на частые вопросы
- собирать заявки
- снижать количество переписок с клиентами

Главный приоритет — не "красота ради красоты", а конверсия (заявки).

---

# Общие принципы разработки

- Пиши максимально просто и понятно
- Избегай переусложнения (overengineering)
- Не используй сложные архитектуры без необходимости
- Предпочитай быстрые решения вместо идеальных
- Код должен быть легко дорабатываемым

---

# Архитектура

Используется:
- Next.js (App Router)
- React (functional components)
- TailwindCSS

Правила:
- Компоненты должны быть переиспользуемыми
- Вся вариативность через props
- Не создавать много похожих компонентов (HeroV2, HeroDark и т.д.)
- Использовать один компонент с параметрами (variant, size и т.д.)

---

# UI/UX принципы

- Минимализм
- Чёткая структура
- Акцент на CTA (кнопки, формы)
- Mobile-first

Каждый блок должен отвечать на вопрос:
"почему клиент должен оставить заявку?"

---

# Основные блоки сайта

Часто используемые секции:

- Hero (первый экран)
- Портфолио / примеры работ
- Как проходит работа
- Цены
- FAQ
- Форма заявки

---

# Поведение AI

- Отвечай на русском языке
- Предлагай простые решения
- Если есть выбор — предлагай более быстрый вариант
- Не предлагай лишние библиотеки без необходимости
- Сначала давай рабочее решение, потом улучшения (если нужно)

---

# Чего избегать

- Сложных state management решений (Redux и т.д.)
- Избыточных абстракций
- "универсальных решений на все случаи жизни"
- больших монолитных компонентов

---

# Приоритет

1. Скорость разработки
2. Простота поддержки
3. Конверсия сайта
4. Чистота кода
